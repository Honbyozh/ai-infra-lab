# 执行与验收规则

## 每日闭环

### 1. 开始前（约 5 分钟）

- 确认当天编号、计划时长、任务和验收项。
- 从 `docs/templates/daily-log.md` 创建当天日志。
- 写下今天最小必须产出的文件或证据。

### 2. 学习与实验

- 周一至周五：总计 60 分钟，优先完成主任务，不用扩展项挤占次日时间。
- 周六：5 个 `50 分钟专注 + 10 分钟休息` 单元。
- 周日：2 小时，完成测试、复盘、证据归档和下周准备。
- 遇到阻塞先记录现象、已尝试步骤和下一步，不用无边界延长学习时间。

### 3. 验收（最后 10～15 分钟）

逐项检查原计划中的“检测与验收”，再检查统一完成定义：

- [ ] 能复现
- [ ] 有证据
- [ ] 能解释
- [ ] 有边界

四项全部满足且当天专属验收通过，状态记为 `通过`。否则记为 `部分完成`、`阻塞` 或 `延后补验`，不能勾选进度表。

### 4. 收尾

- 填写实际用时，不能用计划时长代替。
- 链接所有证据，避免只写“已完成”。
- 更新 `docs/progress.md`。
- 未通过项写明下一次处理日期；受硬件限制的项目同步登记到 `docs/deferred-validations.md`。

## 证据优先级

从强到弱依次为：自动化测试或原始数据、可复现命令输出、代码/配置、截图、文字总结。重要结论尽量保留两类证据。

性能实验至少运行 3 次，保留全部原始数据，以中位数作为主要结果；不得只保留最好的一次。

## 状态定义

| 状态 | 含义 |
|---|---|
| 未开始 | 尚未投入当天任务 |
| 进行中 | 已开始，但尚未执行完整验收 |
| 部分完成 | 有产出，但至少一项验收未通过 |
| 通过 | 当天专属验收与四项统一验收全部通过 |
| 阻塞 | 当前环境或问题使任务无法继续，已有明确证据和后续动作 |
| 延后补验 | 已完成允许的替代验收，但原验收需在指定日期补做 |

## 缺课与顺延

- 不预先将未来日期标为完成。
- 某天无法学习时，在日志中记为缺课并说明原因。
- 默认整体顺延，不把两个完整任务压缩到同一天；若只剩小于 30 分钟的补验，可放入周日复盘。
- 顺延后同时更新进度表中的实际日期，保持 `Wxx-Dx` 编号不变。

## 周日复盘最小内容

- 本周计划时长与实际时长。
- 已通过任务、未通过任务及证据。
- 3 个掌握点、2 个薄弱点、1 个下周优先动作。
- 延后补验和技术债是否有变化。
